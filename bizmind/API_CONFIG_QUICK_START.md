# 🚀 API Configuration - Quick Start Guide

> **快速开始**: 仅需3步配置 ModelScope API！

## 第一步：启用功能

编辑 `.env` 文件（项目根目录）：

```bash
VITE_ENABLE_API_CONFIG=true
```

如果文件不存在，则已存在（项目已创建）。

## 第二步：打开配置面板

**方式1：点击按钮**
- 应用启动后，找到顶部导航栏右侧的 🔧 按钮
- 点击打开配置面板

**方式2：快捷键**
- 按 `Ctrl+Shift+P`
- 立即打开配置面板

**方式3：设置菜单**
- 点击 ⚙️ 按钮
- 选择 "API 配置"

## 第三步：配置您的令牌

### 选项A：使用快速预设（推荐）

1. **点击 "🤖 ModelScope Qwen3" 预设按钮**
   - API 端点自动填充：`https://api-inference.modelscope.cn/v1`
   - 模型自动选择：`Qwen/Qwen3.5-35B-A3B`

2. **粘贴您的令牌**
   ```
   ms-eee04013-7a1a-4ad3-a67a-afaf54ce805c
   ```

3. **点击 "🧪 测试连接"**
   - 等待结果
   - 应该显示：✅ 连接成功！

4. **点击 "💾 保存配置"**
   - 完成！配置已保存

### 选项B：手动配置

1. 在 "API 端点" 字段填入：
   ```
   https://api-inference.modelscope.cn/v1
   ```

2. 在 "API Key" 字段填入：
   ```
   ms-eee04013-7a1a-4ad3-a67a-afaf54ce805c
   ```

3. 在 "文本模型" 下拉菜单选择：
   ```
   Qwen/Qwen3.5-35B-A3B
   ```
   或从列表中选择其他 Qwen3 模型

4. 点击 "🧪 测试连接" → "💾 保存配置"

## 🔍 故障排除

### 问题：看不到 🔧 按钮

**解决方案：**
1. 检查 `.env` 文件是否有：`VITE_ENABLE_API_CONFIG=true`
2. 重启开发服务器：`pnpm tauri dev`
3. 刷新浏览器缓存

### 问题：Ctrl+Shift+P 不打开面板

**解决方案：**
1. 确认 `.env` 启用了功能
2. 检查是否按对了快捷键（不是 Ctrl+P）
3. 确认浏览器没有冲突快捷键

### 问题：连接测试失败 - "API 调用失败"

**解决方案：**
1. 检查令牌是否正确：`ms-eee04013-...`
2. 检查网络连接
3. 验证 API 端点格式是否正确（以 `/v1` 结尾）
4. 打开浏览器控制台（F12）查看详细错误信息

### 问题：配置未保存

**解决方案：**
1. 确保点击了 "💾 保存配置" 按钮
2. 等待 "✅ 配置已保存!" 消息
3. 检查数据库权限：`~/.bizmind/db/bizmind.sqlite`
4. 重启应用

## 💡 常见问题

**Q：多个用户可以使用不同的 API Key 吗？**  
A：当前版本是单用户的。每个用户会覆盖前一个用户的配置。

**Q：可以禁用这个功能吗？**  
A：是的！在 `.env` 文件中设置：`VITE_ENABLE_API_CONFIG=false`

**Q：API Key 是加密的吗？**  
A：当前存储在数据库中是明文。生产环境建议添加文件级或数据库级的加密。

**Q：可以同时配置多个提供商吗？**  
A：当前每次保存会覆盖前一个配置。可以通过配置多份应用副本来实现。

**Q：测试连接会产生费用吗？**  
A：会的，每次测试会调用 API（虽然请求很小）。请检查您的用量。
