## 🎯 根据 Console 日志诊断问题

### 场景 A: 看不到任何日志

**可能原因**:
1. Console 标签没有打开或看错位置
2. 消息输入框为空
3. 应用没有重新加载新代码

**解决方案**:
- 确认 Console 标签是否显示消息
- 检查输入框中是否有文本
- 刷新页面 (F5)，确保最新代码已加载

---

### 场景 B: 看到 `[InputWindow] ⚠️ AI 处理失败, 使用降级方案←`

**原因**: 消息被分到 `misc` 类别，置信度为 0%（AI 未处理）

**诊断**:
1. 往上看 Console，找是否有 `[LLM] ❌` 开头的错误
2. 常见的失败模式:

```
模式 1: [LLM] ❌ invoke 失败: LLM配置未找到
→ 需要在设置中配置 API Key

模式 2: [LLM] ❌ invoke 失败: 今日AI额度已用完
→ 检查 Token 限额

模式 3: [LLM] ❌ invoke 失败: API调用失败: read timeout
→ 网络问题或 LLM 服务响应慢

模式 4: [LLM] ❌ JSON 解析失败
→ LLM 返回格式不符合规范，检查是否有 API 变更
```

---

### 场景 C: 看到 `[LLM] ✅ invoke 调用成功` 但消息仍是 `misc`

**可能原因**: LLM 返回的 JSON 格式有问题

**诊断**:
1. 查看 `[LLM] 解析 JSON..., 前 200 字:` 这一行
2. 看返回内容是否是有效的 JSON
3. 检查是否有必要字段（category、confidence等）
