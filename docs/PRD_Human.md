# BizMind（智脑助手）— 产品需求文档（开发者版）

> **版本**: V1.2 | **日期**: 2026-04-08 | **状态**: Confirmed
> **开发模式**: 一人全栈 + AI辅助编码（VSCode Copilot）

---

## 1. 产品背景与核心问题

### 1.1 目标用户

1-10人小微企业老板/个体工商户。无专职行政、秘书或财务。日常沟通散落在微信（主力）、QQ、邮件、Excel等渠道。身兼CEO、销售、财务、客服，精力严重透支。

### 1.2 核心痛点

老板每天在5+个应用间切换，重要信息淹没在聊天洪流中。没有任何工具能低成本地把这些碎片信息结构化。现有方案（飞书/CRM/Notion）要么需要团队迁移，要么需要手动录入，与小微企业"快、碎、杂"的节奏不匹配。

### 1.3 核心价值主张

BizMind 是一个 **"AI外置大脑"**——不改变用户现有工作方式，只在用户需要时，通过快捷键/截图/粘贴把信息喂给AI，AI负责理解、分类、归档。

**一句话**：不改变你的工作方式，只让信息不再丢失。

| 维度 | 传统方式 | BizMind |
|------|----------|---------|
| 信息收录 | 手动打开CRM/Excel逐条录入 | 快捷键+粘贴，2秒归档 |
| 信息分类 | 人工判断建文件夹 | AI自动识别客户/项目/金额/待办 |
| 项目追踪 | 翻聊天记录找上下文 | 按客户/项目自动聚合时间线 |

### 1.4 竞品定位

| 产品 | 为什么不选 |
|------|-----------|
| 飞书/钉钉 | 需全团队迁移，学习成本高 |
| Notion | 纯手动，无AI归档 |
| CRM类 | 功能冗余，上手门槛高 |
| 微信文件助手 | 无结构化、无检索 |

**BizMind差异点**：极速收录 + AI归档 + 数据本地化（隐私）。

## 2. MVP范围定义（极简版）

### 2.1 开发优先级总览

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0 必须** | 悬浮球 + 快捷键呼出 | 核心入口，没有这个产品不成立 |
| **P0 必须** | 文本粘贴归档 | 最基础的信息收录 |
| **P0 必须** | AI智能分类与实体提取 | 核心价值——自动理解信息 |
| **P0 必须** | 看板视图（单视图） | 信息必须有地方看 |
| **P0 必须** | 本地SQLite存储 | 数据持久化基础 |
| **P0 必须** | 首次启动向导（Onboarding） | 确保用户完成API Key配置，避免首次使用即失败 |
| **P0.5 强烈建议** | 截图粘贴识别（多模态LLM） | 微信截图是高频场景，用户系统截图后Ctrl+V粘贴 |
| **P0.5 强烈建议** | 聊天记录粘贴批量导入 | 冷启动关键——粘贴聊天记录让AI逐条解析 |
| **P0.5 强烈建议** | 全文搜索 | 信息多了之后找不到等于没存 |
| **P1 可延后** | 智能提醒与待办 | 待办提取可做，提醒系统复杂 |
| **P1 可延后** | 自然语言查询（RAG） | 需要向量数据库，复杂度高 |
| **P1 可延后** | 时间线视图 / 表格视图 | 提升体验但不影响核心流程 |
| **P1 可延后** | Mac支持 | MVP先只做Windows |

### 2.2 MVP用户故事

**故事0：首次启动配置**
> 第一次打开BizMind → 弹出欢迎向导 → 选择AI服务商（推荐阿里云Qwen）→ 按引导注册并获取API Key → 粘贴Key并点击"测试连接"→ 验证通过 → 进入主界面。整个过程3分钟内完成。

**故事1：快速归档微信消息**
> 我正在微信里和客户聊报价，选中关键消息复制 → 按Ctrl+Shift+A → 悬浮窗弹出，粘贴文本 → 选择来源"微信" → 点击"归档" → AI自动识别出客户名"张总"、金额"3万"、分类为"客户线索" → 看板多了一张卡片。

**故事2：截图归档聊天记录**
> 微信群里有人发了一段重要讨论 → 我按Win+Shift+S截图框选区域 → 按Ctrl+Shift+A打开悬浮窗 → Ctrl+V粘贴截图 → 图片缩略图出现在输入区 → 点击"归档" → AI通过多模态识别文字并分类归档。

**故事3：批量导入历史聊天记录**
> 首次使用，我从微信电脑版复制与"张总"的近期聊天记录 → 按快捷键打开悬浮窗 → 粘贴大段聊天文本 → 选择"批量导入"模式 → AI解析出每条消息的发送者、时间、内容 → 逐条分类归档（显示进度）→ 立刻看到和张总的沟通记录。

**故事4：回溯项目信息**
> 客户问上次报价多少，我打开BizMind → 搜索"张总 报价" → 立刻找到3周前归档的那条消息 → 确认报价3万。

## 3. 交互流程

### 3.1 首次启动向导

用户首次打开应用（或API Key为空时），自动弹出全屏向导，分步引导：

1. **欢迎页**：产品简介 + "BizMind帮你把碎片信息变成结构化知识"
2. **选择AI服务商**：推荐阿里云Qwen（支持截图识别），同时列出DeepSeek/OpenAI选项
3. **配置API Key**：提供注册链接 + 图文引导 + [测试连接] 按钮验证Key有效性
4. **（可选）导入示例数据**：预置几条示例消息让用户体验看板效果，或跳过
5. **完成**：进入主界面

向导完成后标记 `onboarding_completed = true`，后续启动不再显示。

### 3.2 整体流程

```
日常工作（微信/QQ/邮件）
        ↓
发现需要记录的信息
        ↓
  ┌───────────────────────────────────┐
  │ 方式A: Ctrl+Shift+A              │──→ 悬浮输入窗 → 粘贴文本 → 归档
  │ 方式B: 系统截图后Ctrl+V粘贴图片   │──→ 多模态LLM识别 → 归档
  │ 方式C: 拖拽文件                   │──→ 保存文件引用 → 归档
  │ 方式D: 粘贴聊天记录批量导入       │──→ AI逐条解析 → 批量归档
  └───────────────────────────────────┘
        ↓
  AI处理（≤5秒，显示加载动画）
        ↓
  自动归档 → 看板更新
        ↓
  空闲时：打开看板浏览 / 搜索 / 修正分类
```

### 3.3 悬浮球交互

**收起状态**：
- 半透明圆形图标（48px），停靠屏幕右侧
- 鼠标悬浮显示"BizMind | Ctrl+Shift+A"
- 可拖拽移动
- 右键菜单：打开看板 / 设置 / 退出

**展开状态（输入窗，约400×360px）**：
- 顶部：来源选择Tab（微信 | QQ | 邮件 | 其他）
- 中部：文本输入区（用户手动粘贴内容）；支持Ctrl+V粘贴图片，粘贴后显示缩略图预览
- 底部左侧：[批量导入] 切换按钮
- 底部右侧：[归档] 主按钮 + [归档并标记紧急] 次要按钮
- 归档后：窗口收起，悬浮球闪绿色0.5秒

### 3.4 看板界面

**MVP只做看板视图**（时间线/表格视图V1.0+延后）：
- 顶部：Logo + 搜索栏 + 设置入口
- 主体：水平列（客户线索 | 客户维护 | 项目进度 | 财务单据 | 待办 | 杂项）
- 卡片：来源图标 + 时间 + 摘要（50字）+ 标签（客户/项目/金额）+ 状态点
- 点击卡片：右侧滑出详情面板（完整内容 + AI提取字段可编辑 + 操作按钮）
- 支持拖拽卡片跨列移动

## 4. 风险与简化决策

### 4.1 从原PRD中砍掉/降级的功能

| 原功能 | 决策 | 原因 |
|--------|------|------|
| 本地PaddleOCR引擎 | **砍掉→改用多模态LLM** | 省去50MB+模型打包、跨平台兼容问题，LLM识别质量更高 |
| 后台剪贴板监听+提示条 | **砍掉** | 后台监听侵入性强，用户隐私顾虑大，改为手动操作 |
| 悬浮窗内截图按钮 | **砍掉** | 集成系统截图工具跨平台复杂，用户已熟悉Win+Shift+S后Ctrl+V |
| 微信TXT文件正则解析 | **砍掉→改为AI解析** | 微信导出格式不统一，正则维护成本高，LLM更灵活 |
| API Key AES-256加密 | **砍掉→明文存储** | 一人开发，加解密增加复杂度，明文+本地提示即可 |
| FTS5中文分词优化 | **砍掉** | 优先搜索结构化字段，避免引入分词依赖 |
| Excel审阅助手 | **移到V2.0** | 开发量大，与核心归档价值无关 |
| 文件内容解析 | **降级：仅保存文件+路径** | MVP只记录文件引用，不做内容提取 |
| 自然语言查询（RAG） | **延后到V1.5** | 需要向量数据库，搜索已够用 |
| 语音备忘 | **砍掉** | 桌面端语音输入不自然 |
| 多设备同步 | **延后到V1.5** | 需要云端服务 |
| 团队协作 | **延后到V2.0** | 目标用户是1人老板 |
| 集成扩展 | **延后到V2.0** | 每个集成都是独立工程量 |
| 数据洞察报表 | **延后到V2.0** | 锦上添花 |
| 时间线视图 / 表格视图 | **延后到V1.0+** | 看板视图已满足MVP |

### 4.2 新增的MVP功能

| 功能 | 说明 |
|------|------|
| **首次启动向导** | 引导用户完成API Key配置，提升冷启动成功率 |
| **图片粘贴识别** | 用户系统截图后Ctrl+V粘贴，多模态LLM识别 |
| **批量导入模式** | 粘贴大段聊天记录，AI逐条解析归档 |

### 4.3 性能指标

| 指标 | 要求 |
|------|------|
| 悬浮窗弹出 | ≤500ms |
| AI处理（文本归档） | ≤5s（显示加载动画） |
| AI处理（截图识别） | ≤15s（多模态调用较慢） |
| 看板加载 | 1000条数据下 ≤1s |
| 搜索响应 | ≤500ms（数据量≤5000条） |
| 内存占用 | 后台常驻 ≤150MB |
| 安装包大小 | ≤30MB |

### 4.4 关键风险

| 风险 | 应对 |
|------|------|
| LLM API调用成本失控 | 用户自备Key + 每日Token限额 + 推荐低价模型 |
| 多模态LLM不支持（用户选的纯文本模型） | 降级：提示用户手动输入截图文字 |
| 用户不会获取API Key | 首次启动向导 + 设置页内嵌图文引导 |
| Tauri跨平台兼容问题 | MVP先只做Windows，Mac作为P1 |
| AI分类不准用户放弃 | 分类修正后本地记录，逐步优化Prompt |
| API Key明文泄露 | 设置页提示"仅存储在本机，请勿分享此文件" |

## 6. 商业模式（初步）

| 层级 | 价格 | 权益 |
|------|------|------|
| 免费版 | ¥0 | 每日归档20条，需自备API Key |
| 专业版 | ¥99/月 | 无限归档，含AI额度（或自备Key免此费） |
| 买断版（考虑中） | ¥599一次性 | 永久使用，自备Key |

> 核心思路：软件本身低价甚至免费，AI成本由用户自备Key承担，降低运营压力。

## 7. 成功指标（North Star）

| 指标 | 定义 | MVP目标 |
|------|------|---------|
| 日活跃归档数 | 用户每日归档的信息条数 | ≥5条/天 |
| AI分类免修正率 | 用户未修正AI分类的比例 | ≥75% |
| 7日留存率 | 注册后第7天仍活跃 | ≥30% |
| 冷启动完成率 | 首次启动向导完成率（含API Key验证通过） | ≥60% |
