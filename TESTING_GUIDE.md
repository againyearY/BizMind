## 📝 测试流程

### A. 测试自动刷新功能

**目的**: 验证输入消息后看板是否自动更新

**步骤**:
1. 按 Ctrl+Shift+A 打开快速归档窗
2. 选择来源渠道（推荐选 "微信"）
3. 输入简单文本，如：`李总想咨询我们的ERP项目，预算200万`
4. 点击 "归档" 按钮
5. **预期**: 消息立即出现在看板对应列中（不需要手动刷新）

**查看日志**（Console 中出现）:
```
[InputWindow] 开始调用文本提取
[LLM] 开始调用文本提取
[Commands] 📨 收到 call_llm_text 请求
[LLM Rust] 🔄 开始调用 LLM
[InputWindow] ✅ 消息保存成功
```

### C. 测试删除功能

**目的**: 验证已归档的消息能否被删除

**步骤**:
1. 先用上面的测试输入一条消息
2. 点击卡片打开详情面板（从右侧滑入）
3. 在详情面板下方找到 "删除" 按钮（红色）
4. 点击 "删除" 按钮
5. 系统会弹出确认对话框：`⚠️ 确定要删除这条消息吗？此操作不可撤销。`
6. 点击 "确定" 确认删除
7. **预期**: 
   - 消息立即从看板消失
   - 控制台显示：`[Commands] ✅ 消息已删除`
   - 详情面板自动关闭

**调试**:
- 如果删除失败，详情面板会显示红色错误信息: `删除失败: xxx`
- 检查 Console 中是否有 `[DB] ❌` 错误信息
- 检查是否是权限问题或数据库锁定

---

### D. 在编辑模式下测试更新和删除

**步骤**:
1. 点击卡片打开详情面板
2. 点击 "编辑" 按钮（蓝色）
3. 面板进入编辑模式，所有字段变为可编辑
4. 修改一些字段，如：
   - 分类改为其他类别
   - 客户名改为不同的值
   - 金额修改
5. 点击 "保存" 按钮应用修改
6. **预期**: 
   - 修改立即保存到数据库
   - 看板中的卡片信息更新
   - 控制台显示：`[DetailPanel] ✅ 消息保存成功`

**从编辑模式删除**:
1. 返回查看模式（点击取消）
2. 然后按照第 C 步的流程删除

### 错误场景日志

**错误 1: LLM 配置未找到**
```
[Commands] ❌ LLM 配置未找到
```
→ 需要在设置中配置 API Key

**错误 2: API 调用超时**
```
[LLM Rust] ❌ LLM调用超时(>8秒)
```
→ 网络问题或 LLM 服务响应慢

**错误 3: JSON 解析失败**
```
[LLM] ❌ JSON 解析失败: Unexpected token...
```
→ LLM 返回格式不对，可能 API 发生变化

**错误 4: 删除失败**
```
[Commands] 🗑️ 收到删除请求，消息ID: abc-123
[DB] ❌ 消息不存在，无法删除 (ID: abc-123)
```
→ 消息可能已被删除，或 ID 不匹配

## 🐛 常见问题排查

### Q: 消息没有出现在看板？
**排查步骤**:
1. 打开 F12 Console，查看是否有错误日志
2. 检查消息是否保存到数据库（DebugPanel 中应该显示）
3. 尝试手动刷新页面 (F5)
4. 检查是否是 Vite port 问题（如果 5173 被占用，应该用 5174）

### Q: 消息分类为 misc？
**排查步骤**:
1. 查看 Console 中是否有 `[LLM] ✅ 实体提取成功` 日志
2. 如果没有，查看 `[LLM] ❌` 错误信息
3. 检查输入文本是否符合分类规则
4. 尝试其他测试用例
5. 查看 AI 置信度是否为 0（表示 AI 未处理）

### Q: 删除按钮无反应？
**排查步骤**:
1. 确认是在查看模式（非编辑模式）
2. 确认点击了红色的 "删除" 按钮
3. 查看 Console 中是否有 `[Commands] 🗑️ 收到删除请求`
4. 如果有错误，查询详情面板中显示的错误信息
5. 尝试重启应用

### Q: 修改后数据没有保存？
**排查步骤**:
1. 确认点击了蓝色的 "保存" 按钮
2. 查看 Console 中是否有 `[DetailPanel] ✅ 消息保存成功`
3. 关闭详情面板后重新打开，检查修改是否保存
4. 刷新页面后查看修改是否仍然存在
