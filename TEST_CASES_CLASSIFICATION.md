## ✅ 测试步骤和验证

### 逐个测试流程：

1. **打开应用** - 已运行在 `pnpm tauri dev`
2. **按 Ctrl+Shift+A** - 打开快速归档窗口
3. **选择来源渠道** - 建议选择 "微信" (更接近真实场景)
4. **粘贴测试文本** - 从上面复制用例
5. **点击 "归档" 按钮**
6. **查看结果**：
   - ✅ 消息应该出现在看板对应的列中
   - ✅ 消息卡片应显示正确的客户名/项目名/金额
   - ✅ AI 置信度应该在预期范围内

### 调试技巧 (F12 Console)：

查看日志了解 AI 处理过程：
```
[LLM] 开始调用文本提取，来源: wechat
[LLM] ✅ LLM 响应成功: {...}
[InputWindow] ✅ 消息保存成功: lead
```

如果看不到 `✅ LLM 响应成功`，说明 AI 调用失败，查看错误日志诊断原因。

## 🚨 已知问题和调试

### 问题：消息没被归到正确的分类
- **检查1**: Console 中是否有 `[LLM] ❌ JSON解析失败`？
  - 是 → LLM 返回格式不对，可能 API 故障
  - 否 → 继续检查
- **检查2**: 消息是否进入 `misc` 类别？
  - 是 → 确认 AI 是否调用成功（查看置信度，如果为0说明AI未处理）
  - 否 → 可能分类规则理解有偏差

### 问题：消息输入后没有自动出现
- 已在 2024-04-09 修复
- 消息应该立即出现在对应列的顶部
- 如果没有，刷新浏览器 (F5)

### 问题：置信度显示为 0
- 表示 AI 未成功处理，消息进入降级流程
- 检查 Console 日志查看具体错误
